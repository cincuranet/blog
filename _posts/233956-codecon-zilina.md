Už za dvacet dní, přesně, se bude konat [CODECON #Žilina][1]. Nikdy jsem na této konferenci nebyl, ale Žilina je rozhodně město, které mám rád. A když vidím, na jakých místech se akce koná, nemůžu se dočkat. Moje session je _How I put .NET into Firebird database engine_. Ano, název je v angličtině (protože jsem hlupák a nedošlo mi, že CfP můžu vyplnit v češtině), ale jinak vám vše samozřejmě řeknu v češtině.  

<!-- excerpt -->

![Banner]({{ include "post_ilink" page "banner.png" }})

> Firebird is a database engine similar to MS SQL, Postgres, DB2 and others. Since version 3 it also allows writing plugins. I took this capability and put .NET into it. It allows you to write stored procedures, functions, ... in .NET and have all this code executed in the context (address space, transaction, ...) of database engine. In this session I'll describe what hurdles I had to overcome, how I did it and how I'm hunting best possible performance.
